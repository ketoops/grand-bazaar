🗺 Map Design Notes

The world map is the primary screen. Zones are scattered around Grand Bazaar at varying distances. Closer = earlier zones, further = later zones.

Layout
Hub-and-spoke, scrollable/zoomable
Art Direction
Dark parchment world with warm ambient glow. Each zone has a unique visual biome.
Interaction
Tap zone = open detail. Pinch = zoom. Pan = scroll map.
🐫 Transport System: Caravans

Caravans replace IPM's shuttles. They carry resources from zones to Grand Bazaar along visible trade routes.

Visual Style
Animated sprites moving along dashed paths. Camels for desert zones, horses for European zones, boats for coastal zones.
Upgradeable Stats
Caravan Speed, Cargo Capacity — mirrors IPM's Ship Speed & Cargo per zone.
Future Variants
Ox carts, merchant ships, royal escorts — cosmetic + functional upgrades.

📝 Economy Design Notes
Production Model
Full idle — zones produce 24/7, including offline. No tap-to-collect.
Sell Model
Manual sell at start → auto-sell unlockable via Crowns or Legacy Seals.
Price Model
Fixed base prices per resource + multipliers from upgrades. Dynamic market = future feature.
Resources Per Zone
2 resources per zone from start. Resources can overlap between zones with different yields.
Upgrades Per Zone
4 upgrades: Production Rate, Caravan Speed, Cargo, Multi-Harvest (capped Lv 25).
Scaling Philosophy
Upgrade costs exponential. Later zones have steeper multipliers. Multi-Harvest always ×1.55. Income grows but never catches up to costs — soft walls are intentional.

📋 Loop Design Notes

The core loop follows the proven idle/incremental pattern: earn → spend → reset → earn faster. Key considerations:

Session Length
Target: meaningful progress in 2–5 minute active sessions + idle gains
Reset Pacing
First prestige ~30–60 minutes of total play. Later prestiges speed up.
Engagement Hooks
New zone unlocks, upgrade milestones, visual progression on the world map

📝 Prestige Design Notes
Layer Count
Single layer for v1. One reset type, one prestige currency. Additional layers can be added post-launch if needed.
Formula Design
Square root creates diminishing returns — encourages resetting rather than grinding. The player naturally finds an efficient reset cadence.
Upgrade Structure
Flat list, not a tree. Keeps it simple and easy to balance. Prestige tree is in the Future Systems backlog.
Pacing
First prestige timing: TBD. Design goal: player should want to prestige, never feel forced. Each subsequent run should feel noticeably faster.
Player Psychology
The reset should feel like an exciting new beginning, not a punishment. Show projected Seals before confirming. Show post-reset power increase.

Monetization Planning

High-level revenue model concepts for mobile/tablet

🎬
Rewarded Ads
Optional video ads for temporary boosts
Examples: 2x production for 30 min, instant sell bonus, speed up timers.
Limit: TBD ads per day.
💎
Premium Currency Packs
Purchasable premium currency (name TBD)
Tiers: Small / Medium / Large / Best Value.
Used for: cosmetics, speed-ups, QoL features.
🎨
Cosmetic Themes
Visual skins for zones and Grand Bazaar
Zone themes, map skins, UI color palettes, animated effects.
Purely cosmetic — no gameplay advantage.
🎁
Starter Bundle
One-time new player purchase
Contents: premium currency + starting Crowns boost + exclusive cosmetic.
Discount: ~70% off vs. buying individually.
✨
Quality-of-Life Upgrades
Permanent convenience purchases
Examples: permanent auto-sell, 2x offline earnings, remove interstitial ads, extra manager slots.
One-time purchases — not consumable.
ⓘ Monetization should enhance convenience, not gate core content. The game must be fully playable and enjoyable without spending. Avoid pay-to-win mechanics.
